instruction set for GR8CPU Rev2


The official and final instruction set for GR8CPU revision 2.1.
This goes paired with ASMGV2.1.txt
For your hacking pleasure at events where i bring it.
I fully invite you to somehow pwn it.
In order to fully understand how to program this CPU, you must read the technical description linked below.

Instructions are one byte in size, the upper two bits unused.

Argument types:
VAL - 8 bit signed or unsigned integer
VAL - 8 bit unsigned integer representing an address in memory

Arguments are placed after the instruction byte, the leftmost one coming first.

HEX NAME: SYNTAX
Description.
Arguments:
Arguments...

Instruction set:

00 NOP: NOP
Do nothing.

01 LDR: LOAD ADR
Load from memory.
Arguments:
ADR - address which holds the value to load into A register

02 LDA: LOAD #VAL
Load absolute value.
Arguments:
VAL - value to load into A register

03 CPAC: COPY A TO C
Copy A to C.

04 CPAD: COPY A TO D
Copy A to D.

05 CPCA: COPY C TO A
Copy C to A.

06 CPDA: COPY D TO A
Copy D to A.

07 ADD: ADD ADR
Load from memory and add to A.
Arguments:
ADR - address of value to add to A register

08 ADC: ADD ADR, CI
ADD with carry in on.
Arguments:
ADR - address of value to add to A register

09 ADDA: ADD #VAL
Add absolute value to A.
Arguments:
VAL - value to add to A register

0A ADCA: ADD #VAL, CI
ADDA with carry in on.
Arguments:
VAL - value to add to A register

0B SUB: SUB ADR
Load from memory and subtract from A.
Arguments:
ADR - address of value to subtract from A register

0C SBC: SUB ADR, CI
SUB without carry in on.
Arguments:
ADR - address of value to subtract from A register

0D SUBA: SUB #VAL
Subtract absolute value from A.
Arguments:
VAL - value to subtract from A register

0E SBCA: SUB #VAL, CI
SUBA without carry in on.
Arguments:
VAL - value to subtract from A register

0F CMP: COMP ADR
Compare against ADR in memory witout overwriting A, negative numbers not supported (no 2's complement).
Arguments:
ADR - address of value to compare to A

10 CMPA: COMP #ADR
Compare against absolute value without overwriting A, negative numbers not supported (no 2's complement).
Arguments:
VAL - value to compare against A

11 JMP: JUMP ADR
Jumps to ADR in execution.
Arguments:
ADR - point to continue execution

12 JEQ: JUMP ADR, IF A = B
Jump if 0 flag.
Arguments:
ADR - point to continue execution in case the zero flag is on, A was equal to B

13 JNE: JUMP ADR, UNLESS A = B
Jump unless 0 flag.
Arguments:
ADR - point to continue execution in case the zero flag is off, A was not equal to B

14 JOV: JUMP ADR, IF A > B
Jump if carry and not 0 flags.
Arguments:
ADR - point to continue execution in case carry flag is on and zero flag is off, A would have been larger than B

15 JNO: JUMP ADR, IF A <= B
Jump if not carry or 0 flags.
Arguments:
ADR - point to continue execution in case carry flag is off or zero flag is on, A would have been equal to or smaller than B

16 JUN: JUMP ADR, IF A < B
Jump if neither carry nor 0 flags.
Arguments:
ADR - point to continue execution in case carry flag is off and zero flag is off, A would have been smaller than B

17 JNU: JUMP ADR, IF A >= B
Jump if carry or 0 flags.
Arguments:
ADR - point to continue execution in case carry flag is on or zero flag is on, A would have been equal to or larger than B

18 JCR: JUMP ADR, IF CARRY
Jump if carry flag.
Arguments:
ADR - point to continue execution in case carry flag is on, either from shift / rotate or from the adder IC

19 JRC: JUMP ADR, UNLESS CARRY
Jump unless carry flag.
Arguments:
ADR - point to continue execution in case carry flag is off, either from shift / rotate or from the adder IC

1A ORR: OR ADR
Load from memory and OR with A.
Arguments:
ADR - adress of value to bitwise or with A

1B ORA: OR #VAL
OR A with absolute value.
Arguments:
VAL - value to bitwise or with A

1C AND: AND ADR
Load from memory and AND with A.
Arguments:
ADR - adress of value to bitwise and with A

1D ANDA: AND #VAL
AND A with absolute value.
Arguments:
VAL - value to bitwise and with A

1E XOR: XOR ADR
Load from memory and XOR with A.
Arguments:
ADR - adress of value to bitwise xor with A

1F XORA: XOR #VAL
XOR A with absolute value.
Arguments:
VAL - value to bitwise xor with A

20 LSH: SHIFT L
Shift A left by one (bitwise).

21 RSH: SHIFT R
Shift A right by one (bitwise).

22 STA: STORE ADR
Store a byte into memory at ADR.
Arguments:
ADR - address to place the value of A into

23 INM: INC ADR
Increment a byte in memory, side effect: A is set to the same value as the memory address.
Arguments:
ADR - address to increment the value of

24 INA: INC
Increment A.

25 DCM: DEC ADR
Decrement a byte in memory, side effect: A is set to the same value as the memory address.
Arguments:
ADR - address to decrement the value of

26 DCA: DEC
Decrement A.

27 LSM: SHIFT L, ADR
Shift a byte in memory left, side effect: A is set to the same value as the memory address.
Arguments:
ADR - address to shift the value of

28 RSM: SHIFT R, ADR
Shift a byte in memory right, side effect: A is set to the same value as the memory address.
Arguments:
ADR - address to shift the value of

29 LDO: LOAD ADR, OFFSET A
Load a byte in memory using A as an offset.
Arguments:
ADR

2A ROL: ROT L
Rotate A left.

2B ROR: ROT R
Rotate A right.

2C ROLA: ROT L, ADR
Rotate a byte in memory left, side effect: A is set to the same value as the memory address.
Arguments:
ADR - address to rotate the value of

2D RORA: ROT R, ADR
Rotate a byte in memory right, side effect: A is set to the same value as the memory address.
Arguments:
ADR - address to rotate the value of

2E INCC: INC C
Increment the C register.

2F DECC: DEC C
Decrement the C register.

30 LSC: SHIFT L, C
Shift register C left.

31 RSC: SHIFT R, C
Shift register C right.

32 ROLC: ROT L, C
Rotate register C left.

33 RORC: ROT R, C
Rotate register C right.

...
Invalid and unused instructions, will act as longer NOP, takes no arguments because they do nothing.

3F HLT: HALT
Stops all further execution.
